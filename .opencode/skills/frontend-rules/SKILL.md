# Frontend Development Rules (Flutter & React)

## Flutter (Mobile App)
- **State Management**: Provider / Riverpod.
- **Folder Structure**: Clean Architecture (data, domain, presentation).
- **Naming**: `snake_case` for files, `camelCase` for variables, `PascalCase` for classes.
- **UI**: Follow Material Design 3 guidelines.

## React (Admin Dashboard)
- **Framework**: Functional components + Hooks.
- **Styling**: Vanilla CSS or requested UI library.
- **Naming**: `PascalCase` for components, `camelCase` for hooks/functions.
